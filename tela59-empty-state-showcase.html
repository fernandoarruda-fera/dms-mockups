Empty States

Componente padronizado para estados vazios. Use sempre que uma lista, busca ou tela não tiver conteúdo a exibir — evita silêncio visual e orienta o usuário sobre o próximo passo.

Anatomia

1. ÍconeVisual amigável, 48–64px, cor temática.
2. HeadingCurto, direto. 16–18px semibold.
3. DescriçãoUma frase. Explica o estado e sugere ação.
4. Ação (opcional)CTA primária ou link. Omita se não houver próximo passo.

Faça

  • ✓ Use uma única ação primária quando houver próximo passo claro
  • ✓ Mantenha textos curtos e na voz ativa
  • ✓ Diferencie "vazio inicial" de "sem resultados" — eles pedem ações diferentes
  • ✓ Para erros, mostre código discreto que ajude o suporte

Não faça

  • ✗ Não use ilustrações genéricas de stock — prefira ícones do sistema
  • ✗ Não escreva "ops!" ou frases auto-irônicas em contextos de erro
  • ✗ Não empilhe múltiplos CTAs concorrentes na mesma tela vazia
  • ✗ Não esconda o ícone — ele orienta a leitura visual
